# タイヤサイズ・商品別 売上集計サイト（Version 3）

固定レイアウトの「単品別売上集計リスト」PDFから、商品コード・商品名・タイヤサイズ・売上数量をOCRで読み取り、サイズごとに商品別の販売本数を集計する静的Webアプリです。

## Version 3 の追加機能

- 作業状態をブラウザ内へ保存
- ブラウザを閉じた後でも「保存データを復元」で再編集可能
- 商品名修正だけでなく、サイズ修正・数量修正・削除した行・抽出明細も保存
- PDF集計完了時に自動保存
- 商品名・サイズ・数量の修正後に自動保存
- 行削除後に自動保存
- 手動の「修正内容を保存」ボタンを追加
- 不要になった保存データを削除可能

## 主な機能

- PDFを複数まとめて読み込み
- ページごとにOCR
- `155/65R14` 形式のタイヤサイズを抽出
- 商品コードと商品名を抽出
- 売上数量列を固定位置から取得
- サイズ別・商品別の本数集計
- OCR結果の手修正・行削除
- 修正内容の保存・復元
- 集計CSV／明細CSVの出力
- PDFはブラウザ内で処理し、独自サーバーには送信しない構成

## 起動方法

### GitHub Pages

1. 新しいリポジトリを作成、または現在のリポジトリを開く
2. `index.html`、`styles.css`、`app.js`、`.nojekyll` を上書きアップロード
3. Settings → Pages → `main` / `root` を選択
4. 表示されたURLを開く

### PCで確認

フォルダ内で次を実行し、表示されたURLをブラウザで開きます。

```bash
python -m http.server 8000
```

通常は `http://localhost:8000` です。

## 保存・復元の仕様

保存される内容は次の通りです。

| 内容 | 保存対象 |
|---|---|
| 商品名修正 | 保存されます |
| サイズ修正 | 保存されます |
| 数量修正 | 保存されます |
| 行削除 | 保存されます |
| 抽出明細 | 保存されます |
| 要確認行 | 保存されます |
| 選択したPDFファイル本体 | 保存されません |

保存先はブラウザのローカルストレージです。同じPC・同じブラウザ・同じURLであれば、ブラウザを閉じても復元できます。

注意点：シークレットモード、ブラウザのサイトデータ削除、別PC、別ブラウザ、GitHub PagesのURL変更では保存データが使えない場合があります。

## 使い方

1. PDFを読み込む
2. 「PDFを集計する」を押す
3. 抽出明細で商品名・サイズ・数量を修正する
4. 修正内容は自動保存されます
5. 念のため区切りの良いところで「修正内容を保存」を押す
6. 後日続きから作業するときは「保存データを復元」を押す
7. 最終結果はCSVで出力する

## 商品名辞書

OCRで商品名を誤読した場合は、「抽出明細・修正」の商品名欄を直してください。
商品名の修正内容は商品コード単位でも保存されます。同じPC・同じブラウザでは、次回のPDF読み込み時から修正済みの商品名が自動反映されます。

ブラウザのサイトデータを削除すると、保存した商品名辞書と作業状態は消去されます。

## 読み取り設定

添付サンプルでは、売上数量列の中心がページ幅の約64.3%にあります。帳票レイアウトが同じなら初期設定のままで動作します。

別レイアウトで数量を拾えない場合は、画面の「読み取り設定」で数量列の位置と許容幅を調整します。

## 使用ライブラリ

- PDF.js 3.11.174
- Tesseract.js 5

CDN経由で読み込むため、初回利用時とOCR実行時はインターネット接続が必要です。

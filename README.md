# タイヤサイズ・商品別 売上集計サイト（Version 2）

固定レイアウトの「単品別売上集計リスト」PDFから、商品コード・商品名・タイヤサイズ・売上数量をOCRで読み取り、サイズごとに商品別の販売本数を集計する静的Webアプリです。

## Version 2 の追加機能

- 各サイズの内訳を商品名ごとに表示
- 商品コード単位で同一商品を合算
- 商品名を画面上で修正可能
- 修正した商品名を商品コードに紐づけ、ブラウザへ自動保存
- 次回以降、同じ商品コードは修正済みの商品名を自動使用
- サイズ・商品名・商品コードで検索
- サイズ×商品の集計CSVを出力

## 主な機能

- PDFを複数まとめて読み込み
- ページごとにOCR
- `155/65R14` 形式のタイヤサイズを抽出
- 商品コードと商品名を抽出
- 売上数量列を固定位置から取得
- サイズ別・商品別の本数集計
- OCR結果の手修正・行削除
- 集計CSV／明細CSVの出力
- PDFはブラウザ内で処理し、独自サーバーには送信しない構成

## 起動方法

### GitHub Pages

1. 新しいリポジトリを作成、または現在のリポジトリを開く
2. `index.html`、`styles.css`、`app.js`、`.nojekyll` を上書きアップロード
3. Settings → Pages → `main` / `root` を選択
4. 表示されたURLを開く

### PCで確認

フォルダ内で次を実行し、表示されたURLをブラウザで開きます。

```bash
python -m http.server 8000
```

通常は `http://localhost:8000` です。

## 商品名辞書

OCRで商品名を誤読した場合は、「抽出明細・修正」の商品名欄を直してください。
修正内容は商品コード単位でブラウザのローカルストレージへ保存されます。同じPC・同じブラウザでは、次回のPDF読み込み時から修正済みの商品名が自動反映されます。

ブラウザのサイトデータを削除すると、保存した商品名辞書も消去されます。

## 読み取り設定

添付サンプルでは、売上数量列の中心がページ幅の約64.3%にあります。帳票レイアウトが同じなら初期設定のままで動作します。

別レイアウトで数量を拾えない場合は、画面の「読み取り設定」で数量列の位置と許容幅を調整します。

## 使用ライブラリ

- PDF.js 3.11.174
- Tesseract.js 5

CDN経由で読み込むため、初回利用時とOCR実行時はインターネット接続が必要です。
